# AdPilot 产品需求文档（PRD）

> 版本：v1.1
> 日期：2026-03-17
> 状态：MVP Demo 阶段

---

## 一、产品概述

### 1.1 产品定位

AdPilot 是一款面向 Meta（Facebook）和 TikTok 广告主的**移动端广告管理助手**，计划上架 App Store。解决广告主在移动端快速查看广告数据、管理广告投放的核心场景。

### 1.2 目标用户

跨平台广告主（同时投放 Meta 和 TikTok 广告），需要随时随地监控广告表现、调整预算和开关投放状态的运营人员和决策者。

### 1.3 核心价值

- **统一管理**：一个入口聚合 Meta + TikTok 双平台广告数据
- **移动优先**：为手机端深度优化的交互体验
- **层级下钻**：支持账号 → 广告系列 → 广告组 → 广告的四级穿透

---

## 二、信息架构

```
AdPilot
├── 登录/注册             — 邮箱注册 → 客服绑定（拦截门）
├── 数据（首页）          — 全局数据概览 + 趋势图
├── 广告                  — 四维度广告管理（默认按消耗降序排列）
├── 账号                  — 广告账号管理 + 授权绑定（受配额限制）
├── 设置                  — 账号管理 + 邀请好友 + 退出登录
└── 邀请好友              — 邀请码 + App Store 下载链接 + 额度统计
```

底部导航栏顺序：**数据 → 广告 → 账号 → 设置**。未登录时仅展示登录/注册流程，底部导航栏隐藏。

## 三、功能模块详细描述

### 3.1 数据首页

**功能目标**：让用户一眼掌握整体广告投放状况。

**功能要素**：

- **平台筛选器**：顶部 Tab 切换全部平台 / Meta / TikTok，影响下方所有数据展示
- **日期筛选器**：支持今天、昨天、近7天、近14天、近30天、本月、上月、自定义日期范围
- **核心指标卡片**（2×2 网格）：
  - 总消耗（Spend）
  - 总点击（Clicks）
  - 总展示（Impressions）
  - 总转化（Conversions）
  - 每个卡片包含数值 + 环比变化百分比（上升/下降着色区分）
- **数据更新时间**：实时显示最近一次数据同步时间，带动态绿色呼吸灯
- **数据趋势图**：柱状图，支持 7 个指标 Tab 切换（消耗、点击、展示、CTR、CPC、转化、CPA），柱体上方显示带单位的数据标注

### 3.2 广告管理（核心模块）

**功能目标**：以四个维度（账号、广告系列、广告组、广告）管理广告投放。

**数据层级关系**：

```
广告账号 (Account)
  └── 广告系列 (Campaign)
        └── 广告组 (Ad Set)
              └── 广告 (Ad)
```

**功能要素**：

- **默认排序**：所有维度列表默认按消耗金额降序排列
- **维度 Tab 切换**：顶部四个 Tab（广告账号 / 广告系列 / 广告组 / 广告），点击 Tab 显示该维度的全量扁平列表，Tab 上附带数量角标
- **层级下钻**：点击任一卡片，进入其下级列表。例如点击某广告账号 → 显示该账号下的所有广告系列
- **面包屑导航**：下钻模式下显示路径面包屑，支持点击任一层级回跳
- **返回逻辑**：左上角返回按钮，下钻模式下逐层返回，顶层时回到首页
- **日期筛选**：独立的日期选择器，影响广告数据的消耗、点击、展示、转化等数值
- **数据更新时间**：显示当前数据的刷新时间
- **汇总卡片**：当前列表顶部聚合显示总消耗、总点击、总展示、总转化、CPA
- **搜索**：按名称或 ID 模糊搜索当前维度列表
- **广告卡片**：每张卡片包含：
  - 平台标签（Meta 蓝 / TikTok 红）
  - 名称 + ID
  - 开关状态 Toggle（开启/暂停投放）
  - 数据行：消耗、点击、展示、转化、CPA
  - 日预算展示 + 调整预算按钮
  - 下级提示（如"包含 3 个广告组，点击查看"）
- **预算调整弹窗**（Bottom Sheet）：
  - 显示目标名称
  - 金额输入框（带 $ 前缀）
  - 快捷金额按钮（$50 / $100 / $200 / $500 / $1000）
  - 确认/取消操作

### 3.3 广告账号管理

**功能目标**：管理已绑定的广告账号，支持添加新账号。

**页面布局（从上到下）**：

1. **配额进度条**：Meta / TikTok 各自的已用/总额度进度条
2. **操作按钮区**（紧跟额度下方，始终可见）：
   - 「＋ 添加广告账号」— 触发添加账号弹窗
   - 「🎁 邀请好友，获得更多额度」— 跳转邀请页
3. **账号列表**：每张卡片展示账号头像（平台色）、名称、ID、状态标签（活跃/受限）、本周消耗、广告系列数、CTR
4. **邀请引导 CTA**：当额度使用较多时，底部显示邀请引导

**授权添加**：Bottom Sheet 弹窗，选择 Meta 或 TikTok 平台进行 OAuth 授权绑定，弹窗内显示当前平台已用额度。

### 3.4 登录与注册

**功能目标**：用户通过邮箱注册账号，注册后必须添加客服（微信获客助手或 Telegram 链接）才能进入主界面。

**功能要素**：

- **登录页**：邮箱 + 密码登录，底部「立即注册」跳转
- **注册页**：邮箱 + 密码 + 邀请码（选填），无输入校验（方便快速验收）
- **客服绑定页（注册后拦截门）**：
  - 营销化设计：展示 AdPilot 核心功能亮点（跨平台数据分析、智能预算优化），吸引用户点击添加客服获取使用教程
  - 用户选择「微信添加」或「Telegram 添加」
  - 微信：通过企业微信获客助手外跳（DeepLink 回调）
  - Telegram：通过 t.me 链接外跳（DeepLink 回调）
  - 外跳后页面进入等待状态，模拟 DeepLink 回调后自动进入主界面
- **已有账号登录**：跳过客服绑定，直接进入主界面

### 3.5 账号配额与邀请系统

**功能目标**：通过邀请裂变机制控制用户可绑定的广告账号数量。

**核心规则**：

- 新用户默认每个平台可绑定 **5 个**广告账号（Meta 5 个 + TikTok 5 个）
- 每成功邀请 1 位新用户注册，邀请人和被邀请人各获得 **+5 个**额度（双向奖励）
- 添加新广告账号时进行配额校验，超出配额时提示「额度已满，请邀请好友」并自动导航至邀请页

**邀请页面功能要素**：

- 专属邀请码 + 一键复制
- App Store 应用下载链接 + 一键复制（供分享给好友下载）
- 已邀请人数、获得额度、当前总额度统计
- Meta / TikTok 各平台配额详情

**配额展示位置**：广告账号页顶部、添加账号弹窗内、设置页、邀请页

### 3.6 设置

**功能要素**：

- 用户信息展示（头像、用户名、邮箱、客服绑定状态标签）
- 账号配额概览（Meta / TikTok 进度条）
- 广告账号管理（跳转至账号管理页）
- 邀请好友（跳转至邀请页）
- 退出登录

## 四、交互规范

### 4.1 导航模式

- 未登录时仅展示登录/注册/客服绑定流程，底部导航栏隐藏
- 登录后底部 Tab Bar 固定，四页面平级切换：**数据 → 广告 → 账号 → 设置**
- 广告模块内部支持下钻导航（面包屑 + 返回）
- 所有 Bottom Sheet 弹窗：从底部滑入动画，点击遮罩层或取消按钮关闭

### 4.2 日期选择器

- 首页和广告页各有独立的日期状态
- 共用同一个 Bottom Sheet 弹窗组件，通过 `dateModalTarget` 区分来源
- 选中态使用主色高亮 + 边框
- 自定义日期范围时展开日期输入区域

### 4.3 排序规则

- 广告管理模块所有维度列表（账号/系列/组/广告）默认按消耗金额降序排列
- 搜索结果同样按消耗降序

### 4.4 状态反馈

- 操作反馈统一使用顶部 Toast（深色背景，2秒自动消失）
- 开关切换即时反馈 + Toast 文案
- 预算修改即时反馈 + Toast 显示变更详情

## 六、数据模型

### 6.1 层级数据结构

```javascript
{
  account:  [{ id, name, platform, on, budget, spend, clicks, impressions, conversions }],
  campaign: [{ id, parentId, name, platform, on, budget, spend, clicks, impressions, conversions }],
  adset:    [{ id, parentId, name, platform, on, budget, spend, clicks, impressions, conversions }],
  ad:       [{ id, parentId, name, platform, on, budget, spend, clicks, impressions, conversions }],
}
```

- `parentId` 建立从属关系：campaign.parentId → account.id，以此类推
- `platform` 取值：`'meta'` | `'tiktok'`
- `on` 表示投放开关状态

### 6.2 导航状态

```javascript
navPath: [{ level, parentId, name }]  // 面包屑路径栈
currentDimension: string               // 当前 Tab 选中的维度
isDrillDown: boolean                   // 是否处于下钻模式
```

### 6.3 日期影响

广告页使用日期乘数系统模拟不同时间范围的数据变化：

```javascript
dateMultipliers = {
  today: 0.14, yesterday: 0.15, '7d': 1, '14d': 1.85,
  '30d': 3.6, thisMonth: 2.3, lastMonth: 2.1, custom: 1.5
}
```

消耗、点击、展示、转化等绝对值指标乘以对应系数，CPA 等比率指标保持稳定。

## 七、技术方案

### 7.1 当前实现

- 单文件 HTML（内嵌 CSS + JS），无外部依赖
- 纯原生 JS，无框架
- CSS Custom Properties 实现主题令牌
- `data-action` 属性事件委托机制，零内联事件处理器
- CONFIG 驱动架构（所有预设、指标、标签均为配置对象）
- 集中式 state 对象管理应用状态
- 模块化 JS 层：Utils（纯函数） → Render（DOM 渲染） → Actions（业务逻辑）

### 7.2 后续技术演进建议

| 阶段 | 方向 | 说明 |
|------|------|------|
| Phase 1 | App Store 上架 | 使用 Capacitor/React Native 封装为原生 App |
| Phase 2 | 框架迁移 | 迁移至 React Native，组件化拆分 |
| Phase 3 | 真实数据对接 | 接入 Meta Marketing API + TikTok Marketing API |
| Phase 4 | 状态管理 | 引入 Zustand/Redux 管理全局状态 |
| Phase 5 | 离线/推送 | 离线数据缓存 + Push Notification |
